Where, What, When, and How? (W3H :)

Our aim is to start with a blank sheet and develope a website.

Where to start?

A good knowledge of HTML5, CSS (test yourself and if necessary revise the answers to the questions on this page).

Either build a small framework or use an open source framework such as Bootstrap, see the Bootstrap exercises in the navigation bar.

What to start with?

  1. Decide what the requirements are for your project.
  2. Visit similar websites for ideas.
  3. Don't necessarily trust the sites you visit for their HTML and CSS code, it may well be out of date but do make a note of the layout, navigation, and colour scheme

When to start coding?

Not yet!

Here is a list of tasks you should complete before starting to code your site.

  • Sketch out each page layout. Bear in mind that you will use the Bootstrap layout and grid components to code this. The grid component divides your page layout into rows with upto 12 columns, so see your page layout in terms of rows and columns; for example, header, content and footer rows with the content row having 2 columns.
  • Navigation is your next task. This is likely to be an unordered list with hyperlinks as list items. Write down the navigation hyperlink addresses on your layout sketch.
  • Choose a colour scheme. Read this article and use the tools such as the colour wheel in the article to help choose the colour scheme and make a note of the colour hexidecimal numbers on your layout sketch.
  • Prepare your assets. For a first prototype you may not have the final images; use something similar and scale it to the size it would appear on the page. For other assets, particularly, sound and video, use small sample files to work with.

Create a project with associated folders and files.

An important word on naming convention - BE CONSISTENT!

For example, index.html, about_us.html (all lower case with an _ character to seperate words); similarly for HTML class and id names be consistent, for example, main-container, main-nav, secondary-nav (all lower case with words seperated by a - character).

  1. In Aptana, create a web project and save it as website
  2. In the website project, create folders js, css, graphics, sound, and video
  3. From your Bootstrap exercise project copy the Bootstrap Javascript files bootstrap.js and jquery-latest.js into the js folder (you can use copy & paste in Aptana).
  4. Similarly, copy the Bootstrap CSS file bootstrap.css into the css folder.
  5. You are now ready to test and validate a template file. You can use a copy of this template file to start the coding for each page in your site. Paste the following code and save it in your project as bootstrap_template.html 
    <!DOCTYPE html>
<html lang="en">
   <head>
      <meta charset="utf-8">
      <title>Bootstrap Template</title>
      <link href="css/bootstrap.css" rel="stylesheet"> <!-- Bootstrap CSS -->
      <link href="css/main.css" rel="stylesheet"> <!-- User defined CSS -->
   </head>      
   <body>
   
      <h1>TODO: Bootstrap template - replace with page content</h1>
               
      <script src="js/jquery-latest.js"></script> <!-- JQuery JS -->
      <script src="js/bootstrap.js"></script> <!-- Bootstrap JS -->
      
   </body>
</html>
  6. Validate bootstrap_template.html and drag it into Chrome to check the file.

You are now ready to start coding your site.

How to start coding?

Your sketches for each page should have the layout, navigation and colour scheme details to allow you to start coding without loosing your sanity in the detail of CSS and valid HTML. Go through the following steps for each page, each time copying the bootstrap_template.html code as your starting point.

Let us start with the index page, right-mouse click on bootstrap_template.html file and paste and save it as index.html

  1. Refer to your sketch and use the Bootstrap layout and grid components to code ONLY THE LAYOUT. Leave a TODO comment in each column of each row.
  2. In the Aptana source code window, right-mouse click and select Source > Format. Save the file. Check there are no red circles in the left margin next to the code indicating any errors; and if so, correct any errors.
  3. Before proceeding any further validate your code using the W3C validator and correct any errors. Neither CSS or JS would work without valid HTML code.
  4. Drag the file into Chrome and use the developer tools to inspect the code and satisfy yourself that the display is as you expect it.

Repeat the above steps except this time in step 1 refer to your sketch for the navigation details and the Bootstrap Nav components.

Next, repeat the above steps for each row of the layout except this time in step 1 refer to your sketch for the particular row details and any Bootstrap components as well your own CSS.

Instead of using class and id names declared by Bootstrap, it is good practice to declare your own classes and ids for your custom CSS declarations. Notice, we have linked to a main.css file for the custom CSS code. Being linked after the bootstrap.css means it will override any Bootstrap declaration.

Design and development is cyclical and you are likely to repeat the above (making modifications and even major changes) before finalising your site. The preparation work, validation, and being systematic is essential and does save time!
